# PoE2 トレード検索URL自動生成ガイド (AI対応)

このドキュメントは、AIがPath of Exile 2（PoE2）の公式トレードAPIと通信し、正しいアイテム検索URLを生成するための仕様と手順をまとめたものです。
AIにこのドキュメントの内容を読み込ませる（コンテキストとして与える）ことで、即座に任意のアイテム条件に合致するURLを作成させることができます。

---

## 🚀 PoE2 トレードAPIの動作原則

PoE2のトレードサイトは、単純なクエリパラメータ（`q=JSON`など）によるWeb検索を公式にはサポートしていません。
動作する正しい検索URLを生成するには、以下の2ステップのAPI連携が必要です。

### 1. 検索条件JSONを公式APIにPOST送信する
*   **エンドポイント**: `https://www.pathofexile.com/api/trade2/search/{League}`
*   **必要なヘッダー**:
    *   `Content-Type: application/json`
    *   `User-Agent: PoE2TradeHelper/1.0 (Contact: mail@example.com)` (公式APIポリシーにより必須)
    *   `Cookie: POESESSID={Your_POESESSID}` (複雑な検索を許可するために必須)
*   **リクエストボディ**: アイテム条件を記述したJSONクエリ

### 2. レスポンス内のIDから検索URLを構築する
APIが返却する `"id"`（Search ID）を使用して、以下のURLを組み立てます。
*   **完成URL**: `https://www.pathofexile.com/trade2/search/{League}/{SearchId}`

## 🔑 アカウントセッションID（POESESSID）の手動設定方法

ゲスト状態（未ログイン）で多数のModや複雑なフィルタをAPIに送信すると、**「Query is too complex.（クエリが複雑すぎます）」** というエラーで弾かれます。
これをお使いの **個人アカウント** でログインした状態で実行することで、複雑なクエリの送信制限が解除されます。

ブラウザのデベロッパーツール等を使って手動で `POESESSID` を取得し、`.env` ファイルに保存してください。

### 手動取得と設定の手順
1. **公式サイトにログインする**:
   お使いのWebブラウザ（Chrome, Edge, Firefox等）で [Path of Exile 公式サイト](https://www.pathofexile.com/) にアクセスし、ログインします。
2. **デベロッパーツールを開く**:
   キーボードの `F12` キー（または `Ctrl + Shift + I` / Macは `Cmd + Option + I`）を押してデベロッパーツールを起動します。
3. **Cookieの値をコピーする**:
   * **Chrome / Edgeの場合**: 
     デベロッパーツールのタブから **「アプリケーション (Application)」** (表示されていない場合は `>>` アイコンをクリック) -> 左側サイドバーの **「ストレージ (Storage)」** 内にある **「Cookie」** -> `https://www.pathofexile.com` を選択します。
     一覧の中から名前が **`POESESSID`** になっている行を探し、その **「値 (Value)」** （32文字の英数字）をダブルクリックしてコピーします。
   * **Firefoxの場合**:
     デベロッパーツールのタブから **「ストレージ (Storage)」** -> 左側サイドバーの **「Cookie」** -> `https://www.pathofexile.com` を選択し、**`POESESSID`** の値をコピーします。
4. **`.env` ファイルに設定する**:
   `apps/97_poe/` フォルダの直下に `.env` ファイルを作成（または編集）し、以下のようにコピーしたセッションIDを記述して保存します。
   ```env
   POESESSID=あなたのPOESESSIDの値（例: 939aeec293633905298fcd42727572a9）
   ```

> [!NOTE]
> ### 📅 セッションID（POESESSID）の寿命について
> このセッションIDは、**一度取得すれば基本的に数日から数週間（最大数ヶ月）有効**です。ブラウザを閉じたりPCを再起動しても有効期限は切れません。
> ただし、以下の場合はセッションが無効化（リセット）されます：
> *   公式サイトで明示的に「ログアウト（Sign Out）」をクリックしたとき
> *   他のPCや異なるブラウザから新しくログインしたとき
> *   公式サーバーのメンテナンス等でセッションがクリアされたとき
> 
> *※もしAIが出力した検索コマンドで「Query is too complex.」や「Forbidden」のエラーが出るようになった場合は、有効期限切れの合図ですので、ブラウザから新しい `POESESSID` を取得し、`.env` ファイルを更新してください。*

## ⚡ AIの負荷と出力を削減するためのプログラム化

クエリの構築とAPI送信のロジックは、ローカルのPythonスクリプト **`generate_trade_url.py`** にカプセル化（プログラム化）されています。
このスクリプトは、自動回収された `.env` ファイル内の `POESESSID` を自動的に読み込みます。

### AIへの命令プロンプト例（AIの負荷を下げる方法）
このファイルがコンテキストにあれば、ユーザーから「冷気フラットとライフ50以上の指輪のURLを作って」と言われた際、AIは**以下の1行を出力するだけで済みます**。

```bash
# AIは以下のコマンドを提示するだけでURLが自動生成されます
python generate_trade_url.py --type Ring --life 50 --cold --league "Runes of Aldur"
```

#### スクリプトのパラメータ引数仕様
*   `--type`: アイテムタイプ (デフォルト: `"Ring"`)
*   `--life`: 必要最小ライフ (数値)
*   `--res`: 必要最小属性耐性合計 (数値)
*   `--phys`: 物理フラット追加ダメージを検索対象にする (フラグ)
*   `--cold`: 冷気フラット追加ダメージを検索対象にする (フラグ)
*   `--cast`: キャスト速度を検索対象にする (フラグ)
*   `--league`: 対象リーグ (デフォルト: `"Runes of Aldur"`)
*   `--session`: アカウントの `POESESSID` (デフォルトで環境変数 `POESESSID` または `.env` ファイルから自動取得)
*   `--max-price`: 予算上限（数値、Chaos Orb単位、デフォルトは無制限）
*   `--any-status`: オフラインの出品者も含める (フラグ、デフォルトはオンラインのみ)
*   `--instant-only`: インスタントバイアウト（ゲーム内自動取引所での即時購入）のみに制限する (フラグ、デフォルトは制限なし)

## 📋 AI向け出力フォーマット・プロンプト定義

AIがユーザーから「ビルドサイトの情報」や「自然言語による装備の検索依頼」を受け取った際は、以下の手順とルールに従って、必ず指定された定型フォーマットで結果を出力してください。

### 🧭 AIの処理ステップとルール

1. **検索要件と取引仕様の整理**:
   ユーザーの入力（現在のレベル、ビルドの方向性、欲しい能力など）から、必要なステータス（ライフ、属性耐性、攻撃力など）を整理します。また、以下の仕様を必ず整理してください。
   * **予算 (Max Price)**: 予算上限があるかどうか。
   * **取引ステータス**: オンラインプレイヤーのみ（Online Only）か、オフライン含むか（Any）。
   * **購入形態**: インスタントバイアウト（Instant Buyout）限定か、通常トレード含むか。
2. **【重要】曖昧な場合の聞き返しルール**:
   * 以下の項目がユーザーの入力から読み取れず、判断に迷う場合は**勝手に仮定してURLを作る前に、ユーザーに質問・確認してください**。
     * 「予算上限はあるか（Chaos Orb等の希望額）」
     * 「通常トレード（オンライン）でよいか、それともゲーム内自動取引所での即時購入（Instant Buyout）に絞るか」
     * 「狙いたい特定のベースアイテムや、特に優先して改善したい部位はあるか」
   * ※ 質問する際は簡潔に選択肢を示すなど、答えやすい形で聞き返してください。
3. **URLの生成（コマンドの実行）**:
   整理した条件に基づき、`apps/97_poe/` 配下にある `generate_trade_url.py` に適切なオプション（`--max-price`, `--any-status`, `--instant-only` など）を付与して実行（`run_command`ツールを使用）し、実際のPoE2公式トレード検索URLを生成します。
4. **定型フォーマットでの出力**:
   生成されたURLを元に、以下のマークダウン形式でユーザーに回答を提示します。
